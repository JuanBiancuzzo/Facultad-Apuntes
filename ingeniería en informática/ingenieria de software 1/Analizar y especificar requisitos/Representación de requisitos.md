# Definición
---
Hay varias alternativas a la representación de los [[ingeniería en informática/ingenieria de software 1/Ingeniería de requisitos/Requisito|requisitos]]
* El lenguaje natural: es el más habitual pero este puede ser ambiguo produciendo conflictos que el [[ingeniería en informática/ingenieria de software 1/Analizar y especificar requisitos/Análisis de requisitos|análisis]] quiere evitar
* Modelos visuales: donde se muestra la relaciones entre los datos, flujos lógicos, etc
* Especificación formal: se describen mediante el uso de lenguajes matemáticos muy precisos
